Arquitetura do Sistema

Backend

  • Linguagens: Node.js (NestJS) ou GoLang
  • Banco de Dados: PostgreSQL, MongoDB/DynamoDB, Redis
  • Mensageria: RabbitMQ/Kafka
  • Armazenamento: S3/GCS
  • Autenticação: JWT, OAuth2, OpenID

Frontend

  • Framework: Angular com NgRx
  • UI/UX: TailwindCSS
  • Componentes: Integração com API em tempo real
  • Visualização: Dashboards interativos
  • Responsividade: Mobile-first design

Infraestrutura

  • Contêineres: Kubernetes
  • API Gateway: Nginx/Kong
  • Load Balancer: Distribuição de carga
  • Monitoramento: Prometheus + Grafana
  • CI/CD: Pipeline automatizado

Segurança

  • Criptografia: End-to-end
  • Compliance: LGPD/GDPR
  • Auditoria: Logs de acesso
  • Backups: Automáticos e redundantes
  • Testes: Penetração e vulnerabilidade

Infraestrutura de Servidores

Amazon Web Services (AWS)

  • EC2: Instâncias t3.medium para aplicação
  • RDS: PostgreSQL para banco de dados relacional
  • ElastiCache: Redis para cache e sessões
  • S3: Armazenamento de imagens e backups
  • CloudFront: CDN para distribuição de conteúdo
Escolha recomendada

Google Cloud Platform (GCP)

  • Compute Engine: VMs e2-standard-2
  • Cloud SQL: PostgreSQL gerenciado
  • Memorystore: Redis para cache
  • Cloud Storage: Armazenamento de objetos
  • Cloud CDN: Distribuição de conteúdo
Alternativa viável

Digital Ocean

  • Droplets: Standard 4GB/2CPU
  • Managed Databases: PostgreSQL
  • Managed Redis: Cache e sessões
  • Spaces: Armazenamento compatível com S3
  • CDN: Distribuição de conteúdo global
Opção econômica

Justificativa da Escolha: AWS

Para o projeto de Monitoramento de Presença Escolar, recomendamos a AWS como plataforma de infraestrutura devido à sua robustez, escalabilidade e conjunto completo de serviços que atendem às necessidades específicas do projeto:

  • Segurança avançada: Essencial para um sistema que lida com dados sensíveis de estudantes, a AWS oferece recursos de segurança de nível empresarial e conformidade com LGPD/GDPR.
  • Escalabilidade: Capacidade de escalar horizontalmente para atender ao crescimento do número de escolas no sistema.
  • Serviços especializados: Acesso a serviços como Amazon Rekognition para reconhecimento facial e Amazon Kinesis para processamento de streams de vídeo em tempo real.
  • Presença global: Infraestrutura distribuída que permite baixa latência para escolas em diferentes regiões.

Dinâmica de Equipe

Princípios de Trabalho

  • Comunicação Transparente

    Todos os membros da equipe devem manter comunicação clara e direta, compartilhando progresso e impedimentos nas dailies.

  • Autonomia com Responsabilidade

    Cada membro tem autonomia para tomar decisões técnicas em sua área, mas deve considerar o impacto no projeto como um todo.

  • Revisão de Código

    Todo código deve passar por revisão de pelo menos um outro desenvolvedor antes de ser integrado.

Rituais da Equipe

  • Daily Standup (15 min)

    Reunião diária às 9:30 para alinhamento de progresso, impedimentos e planos para o dia.

  • Sprint Planning (2h)

    Planejamento quinzenal para definir escopo da sprint, estimar esforço e alocar tarefas.

  • Retrospectiva (1h)

    Ao final de cada sprint para identificar pontos de melhoria e celebrar conquistas.

Metodologia e Importância do Projeto

Importância do Projeto

Segurança Escolar Aprimorada

O monitoramento de presença em tempo real permite identificar rapidamente ausências não justificadas, contribuindo para a segurança dos alunos e tranquilidade dos pais. Em casos de emergência, o sistema fornece informações precisas sobre quem está presente no ambiente escolar.

Otimização de Recursos Educacionais

Com dados precisos sobre frequência, as instituições podem otimizar a alocação de recursos, identificar padrões de ausência e implementar intervenções pedagógicas direcionadas, melhorando os resultados educacionais e reduzindo a evasão escolar.

Transformação Digital da Educação

O projeto representa um passo importante na transformação digital do ambiente educacional, integrando tecnologias avançadas de reconhecimento e análise de dados para criar um ecossistema educacional mais eficiente, seguro e adaptado às necessidades do século XXI.

Metodologia de Levantamento de Requisitos

Workshops Colaborativos

Sessões imersivas com stakeholders para identificar necessidades, expectativas e casos de uso prioritários. Utilizamos técnicas de Design Thinking e User Story Mapping para capturar requisitos de forma colaborativa e centrada no usuário.

4 workshops planejados

Mentorias Especializadas

Sessões de mentoria com especialistas em educação, segurança e tecnologia para validar abordagens e garantir que o sistema atenda às melhores práticas e normas regulatórias do setor educacional.

Mentoria contínua

Ciclos de Validação

Processo iterativo de validação de requisitos com protótipos e MVPs para garantir alinhamento com as necessidades reais dos usuários. Feedback contínuo é incorporado ao desenvolvimento.

Sprints de 2 semanas

Processo de Levantamento e Gestão de Requisitos

Imersão e Descoberta

Entrevistas com gestores escolares, professores, pais e alunos para compreender o contexto, desafios e oportunidades. Análise de sistemas existentes e benchmarking de soluções similares.

Definição e Priorização

Workshop de priorização com stakeholders para definir requisitos essenciais (MVP), importantes e desejáveis. Criação de user stories, critérios de aceitação e definição de DoD (Definition of Done).

Prototipagem e Validação

Criação de protótipos interativos para validação de fluxos e interfaces. Testes de usabilidade com usuários reais e coleta de feedback estruturado para refinamento dos requisitos.

Documentação e Gestão

Documentação detalhada de requisitos funcionais e não-funcionais, integrações, fluxos de dados e regras de negócio. Implementação de processo ágil para gestão de mudanças nos requisitos.

Considerações do Cronograma

  • Paralelismo: Algumas atividades ocorrem simultaneamente para otimizar o tempo de entrega.
  • Buffers: Períodos de contingência estão incorporados para absorver atrasos inesperados.
  • Abordagem Iterativa: Desenvolvimento em sprints de 2 semanas com entregas incrementais.
  • Revisões: Pontos de revisão e aprovação ao final de cada fase para garantir qualidade.
  • Expansão Gradual: Implementação em ondas para permitir ajustes com base no feedback.

Roadmap do Projeto

Fase 1: Planejamento

Duração: 1 mês

  • Levantamento de requisitos com stakeholders
  • Definição das APIs para comunicação com as câmeras
  • Modelagem do banco de dados e arquitetura
  • Setup de infraestrutura inicial

Fase 2: Desenvolvimento do Core

Duração: 2 meses

  • Backend: Autenticação, APIs, banco de dados
  • Frontend: Design e primeira versão das telas
  • Integração com as câmeras e recebimento de eventos
  • Implementação do sistema de notificações

Fase 3: Testes e Otimização

Duração: 1,5 mês

  • Testes de carga e segurança
  • Refinamento do frontend e ajustes de UX/UI
  • Correções de bugs e melhorias de performance
  • Otimização de algoritmos de reconhecimento

Fase 4: Piloto e Ajustes Finais

Duração: 1,5 mês

  • Implementação em escolas piloto
  • Coleta de feedbacks e ajustes finais
  • Preparação para escalabilidade
  • Treinamento da equipe de suporte e documentação
